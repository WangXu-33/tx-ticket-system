# Scripts 目录说明

## 用途

`scripts` 目录现在只保留历史库升级需要的脚本。

- 全新初始化数据库：使用项目根目录的 `init.sql`
- 旧库补齐历史结构变更：使用本目录的 `schema_upgrade_all.sql`

## 文件

- `schema_upgrade_all.sql`
  按历史时间顺序合并后的升级脚本，覆盖：
  - 用户扩展字段
  - 权限模型拆分
  - 头像文件引用
  - 数据权限与索引补齐
  - 权限分析与代码生成菜单补齐
  - 公告消息表结构

## 说明

1. 这是升级脚本，不是全量建库脚本。
2. 如果数据库已经是最新结构，不需要重复执行。
3. 如果是全新环境，优先执行项目根目录的 `init.sql`。
4. 目录中的 Python 临时脚本已经移除，避免继续误用本地一次性工具。
