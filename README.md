# 头绪工单系统

头绪工单系统 是一个基于 `Spring Boot 3 + Sa-Token + MyBatis-Plus + Vue 3 + Vite + Ant Design Vue` 的权限管理基础框架，目标是提供一套可直接继续业务开发的 RBAC 底座，而不是只停留在登录和菜单演示。

## 系统能做什么

- 提供账号登录、会话校验、菜单权限、按钮权限等基础认证授权能力
- 提供用户、角色、菜单、部门四类核心 RBAC 管理能力
- 提供字典、存储配置、附件管理、仪表盘、操作日志、登录日志等系统级模块
- 提供本地存储、阿里云 OSS、MinIO 的统一文件存储抽象
- 提供图片、PDF 的统一文件预览入口，其他附件提供下载兜底
- 提供 Web 管理端，当前版本不包含小程序/移动端代码

## 快速启动

### 1. 初始化数据库

- 新环境直接执行根目录的 [init.sql](/E:/code/gemini/个人工单系统/init.sql)
- 本地 Windows 环境可运行 `scripts/init_database.ps1 -User root -Password <你的密码>`

### 1.1 已有数据库升级

- 如果你本地已经跑过旧版本库，不要只替换代码
- 执行 [scripts/schema_upgrade_all.sql](/E:/code/gemini/个人工单系统/scripts/schema_upgrade_all.sql)
- 这一步会补齐用户表字段、修正字典表时间列、补权限表备注列，并增加当前查询需要的索引
- 第二个补丁会把按钮权限模型从“权限表直接挂菜单”升级为“全局权限定义 + 菜单挂载关系 + 用户直授权预留”

### 2. 启动后端

- 进入 `rbac-java`
- 修改 `src/main/resources/application.yml` 中的数据库连接
- 使用 JDK 21 启动 Spring Boot 服务
- 默认端口为 `8080`

### 3. 启动前端

- 进入 `rbac-vue`
- 安装依赖
- 执行 `pnpm dev`
- 默认开发端口为 `3000`
- 本地 Windows 环境可运行 `scripts/build_frontend.ps1`，脚本会优先使用 `E:\env\node\nvm\v20.18.1`
- 文件预览当前支持图片、PDF 和下载兜底，不再依赖 `@vue-office/*`

## 当前修复重点

当前阶段优先处理“已完成模块的稳定性问题”，不继续扩展代码生成器。最近已处理的重点包括：

- 本地存储静态资源映射不再写死路径
- 个人中心头像上传改为走登录可用的通用上传接口
- 个人中心增加当前登录人资料接口，避免依赖用户管理权限
- 代码生成页改为稳定占位，避免前端构建失败
- 初始化脚本与现库迁移脚本已补齐用户表、字典表、权限表的结构对齐
- 按钮权限已升级为可跨菜单复用，并在角色授权侧加入独立分页授权能力
- 文件预览/下载已改为按文件 ID 访问，支持头像正式文件引用、附件管理图片缩略图、当前页弹窗预览
- 附件上传安全已支持后台配置，位置：系统管理 → 存储配置 → 本地存储 → 附件安全配置
- 文件管理已补齐统计报表、导出和 `ONLY_FULL_GROUP_BY` 兼容修复
- 菜单、角色、部门页已补充树搜索入口，并沉淀可复用的树搜索组件与后端树构建工具
- Web 管理端已抽取统一主题配色，后续改主色、状态色优先修改主题变量
