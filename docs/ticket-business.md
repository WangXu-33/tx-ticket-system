# 头绪工单系统业务说明

## 模块范围

头绪工单系统当前 MVP 包含客户注册、工单提交、工单处理、转派、流程记录、知识库维护和知识沉淀。系统英文名为 TX Ticket System，业务表统一使用 `tx_` 前缀。

## 角色建议

- 客户：注册后提交工单、查看自己的工单、补充资料、确认关闭。
- 运维人员：接单、回复、转派、解决工单。
- 主管：查看团队工单、分派工单、维护和发布知识库。
- 管理员：管理角色、菜单、权限、字典和系统配置。

## 工单流程

1. 客户在“我的工单”提交问题。
2. 系统生成 `TXyyyyMMddHHmmssSSS` 格式工单编号。
3. 运维在“工单工作台”接单或分派处理人。
4. 处理过程中每次回复、转派、解决、关闭都会写入 `tx_ticket_flow`。
5. 已解决工单可以沉淀为知识库草稿。

## 状态编码

- `pending`：待受理。
- `processing`：处理中。
- `waiting_customer`：待客户补充。
- `transferred`：已转派。
- `resolved`：已解决。
- `closed`：已关闭。
- `rejected`：已驳回。

## 知识库 Agent 预留

知识库文章已经结构化保存问题现象、原因分析、解决步骤、适用范围、标签和来源工单。后续接入 Agent 时，建议优先使用已发布文章作为检索源，并在回答中展示来源知识库和关联工单，避免无依据输出。

## 数据安全约束

工单业务不执行物理删除。需要废弃数据时，应使用逻辑删除字段或状态控制；批量覆盖场景必须先归档历史数据。
